    Linear Programming

    2 unknowns x1 and x2 each constrain specifies a "halfplane" that contain(x1, x2)

    define "feasible set" = intersection of all half planes

    possible: * bounded polygon * unbounded polygone * empty

    if polygon is not empty, then it is a convex set. if solution to LP - max C^Tx, Ax <=b exisists, then it must exist at a corner, or sometimes a whole edge in rare cases

    Network

    Given n users of network, how should we route traffic? Goals: meet the minimum user bandwidth requirement, respect the bandwidth limits, maximize your profit given the dollars per bandwidth unit

    X_ab is the total bandwidth from A to B along short path X'_ab is the total bandwidth from A to B along the long path

    Any LP can be written in standard form, maximizing C^T x and Ax <= b, x>= 0

    1. converting min C^Tx to max(-c)^Tx
    2. converting a^Tx >= b to -a^T <= -b
    3. convert a^Tx = b into -a^T <= -b, a^T <= b
    4. if x can be bothpositive and negative, x = x1 - x2, both of which are positive

    The standard form is where all variables are nonnegative, the constraints are all equations, and the objective is to be minimized

    Algorithm Simplex

    find some vertex in the feeasible set, check if some neighboring vertex has a higher value of c^T*x, if found, move there and repeat, else found the optimum
